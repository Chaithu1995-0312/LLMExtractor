# Appendix

## Glossary

| Term | Definition |
|------|------------|
| **Brick** | The smallest atomic unit of information in Nexus. Typically a single message or paragraph from a conversation. |
| **Wall** | A storage aggregation of multiple processed conversation trees. Originally intended to mimic "Message Walls". |
| **Conversation Tree** | The JSON representation of a single chat session, extracted from the monolithic dump. |
| **Cognition Artifact** | A structured JSON file generated by the Assembler, representing a synthesized answer to a topic query. |
| **Anchor** | A user-defined override (Promote/Reject) applied to a specific Brick to influence future recall for similar queries. |
| **Sync** | The process of ingesting `conversations.json`, splitting trees, extracting bricks, and building the vector index. |
| **Recall** | The process of finding relevant Bricks using Vector Similarity + Reranking. |
| **Provenance** | The chain of custody linking a Fact in an Artifact back to the specific Brick and Source File it came from. |

## File Formats

### 1. Brick Metadata (`brick_ids.json`)
```json
[
  "brick_id_1",
  "brick_id_2",
  ...
]
```
*Note: The actual metadata map is managed by `BrickStore`, which reads the source files directly via the ID.*

### 2. Graph Node (`nodes.json`)
```json
{
  "id": "string",
  "type": "artifact|topic|brick",
  "data": { ... }
}
```

### 3. Cognition Artifact (`artifacts/*.json`)
```json
{
  "artifact_id": "sha256...",
  "created_at": "ISO8601",
  "query": "string",
  "payload": {
    "topic": "string",
    "provenance": { ... },
    "raw_excerpts": [ ... ],
    "coverage_status": "ASSEMBLED"
  }
}
```

## System Environment
- **Language**: Python 3.8+
- **Search**: FAISS (CPU)
- **Embeddings**: SentenceTransformers (`all-MiniLM-L6-v2` inferred)
- **Web Server**: Flask
- **Frontend**: React (Vite)
